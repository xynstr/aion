# AION Maintenance Guide

> This file documents **where exactly** things need to be updated when changes are made to AION —
> for use by Claude Code and by AION itself.
> AION reads this via `file_read('MAINTENANCE.md')` or `lookup_rule('maintenance')`.
>
> **Rule:** Before any change, find your change type below and work through the checklist.

---

## Adding a New Tool / Plugin

- [ ] `plugins/{name}/{name}.py` — `register_tool()` with `description`, `input_schema`, `tier`
- [ ] `plugins/{name}/README.md` — **first line = plugin summary** (injected into system prompt automatically!)
- [ ] `AION_SELF.md` — add tool under the correct plugin section (include parameters)
- [ ] `CHANGELOG.md` — entry under today's date
- [ ] `prompts/rules.md` — add behavior rule if tool requires special usage patterns
- [ ] The **Capability Index** in the system prompt updates automatically — no manual step needed

---

## Modifying an Existing Tool (parameters, behavior, name)

- [ ] Plugin file — update `func` + `input_schema`
- [ ] `plugins/{name}/README.md` — update parameter table
- [ ] `AION_SELF.md` — update tool description
- [ ] `CHANGELOG.md` — entry
- [ ] On rename: check `prompts/rules.md` for old name and update

---

## Changing Behavior Rules (how AION should act)

- [ ] `prompts/rules.md` — add or modify rule
- [ ] `AION_SELF.md` — update affected behavior section if relevant
- [ ] `CHANGELOG.md` — entry
- [ ] Invalidate system prompt cache: `self_restart` or restart AION

---

## Changing Core Architecture (aion.py, aion_session.py, plugin_loader.py, aion_web.py)

- [ ] Code change
- [ ] `AION_SELF.md` — update the affected architecture section
- [ ] Regenerate summary: call `generate_self_doc_summary()`
- [ ] `CHANGELOG.md` — entry
- [ ] `prompts/rules.md` — if AION's behavior toward the user changes
- [ ] Changes to `aion.py` itself require a full restart (no hot-reload)

---

## Adding a New Channel / Provider

- [ ] Create provider plugin in `plugins/{provider_name}/`
- [ ] `docs/` — add or update provider documentation
- [ ] `AION_SELF.md` — add provider section
- [ ] `CHANGELOG.md` — entry
- [ ] `prompts/rules.md` — if the channel requires special behavior (like Telegram)

---

## Changing System Prompt / rules.md

- [ ] `prompts/rules.md` — make the change
- [ ] Invalidate system prompt cache: `self_restart`
- [ ] Verify: does AION behave as expected after restart?
- [ ] Check `max_rules_chars` in `config.json` (currently 35000) — ensure file fits

---

## Capability Index (tool overview in system prompt)

**No manual step needed.** The index is auto-generated by `_build_capability_index()` in `aion.py`:
- Groups all registered tools by name prefix (e.g. `desktop_*` → group DESKTOP)
- Rebuilt on every start and after `self_restart`
- Always reflects the current plugin state

## Self-Improvement Systems

| System | Trigger | File | Code location |
|--------|---------|------|---------------|
| **Mistakes Journal** | AION calls `record_mistake()` | `mistakes.md` | `plugins/core_tools/core_tools.py` |
| **Mistakes Injection** | Every session start | *(dynamic)* | `aion.py → _get_mistakes_hint()` |
| **Doc Freshness Warning** | Every session start | *(dynamic)* | `aion.py → _get_doc_freshness_hint()` |
| **Boot Maintenance Session** | Every AION restart (if >1h offline) | `last_boot.txt` | `plugins/boot_session/boot_session.py` |
| **Offline Duration Hint** | Every session start | *(dynamic)* | `aion.py → _get_offline_hint()` |
| **End-of-Conversation Protocol** | User says bye/done | *(rules)* | `prompts/rules.md` |

## Runtime Files (gitignored — personal, not versioned)

| File | Purpose | Created by |
|------|---------|------------|
| `mistakes.md` | Persistent mistake log, injected at session start | `record_mistake` tool |
| `last_boot.txt` | Timestamp of last AION boot | `boot_session` plugin |
| `thoughts.md` | AION's inner monologue journal | `reflect` tool |
| `character.md` | AION personality + user knowledge | `update_character` tool |
| `aion_memory.json` | Persistent memory entries | memory system |
| `todo.md` | Personal task backlog | `todo_add` tool |
| `config.json` | Runtime configuration | web UI / tools |
| `conversation_history.jsonl` | Full conversation log | auto |

## Quick Reference: Key Files

| File | Purpose |
|------|---------|
| `aion.py` | Core: dispatch, system prompt, tool schemas, self-improvement hooks |
| `aion_session.py` | LLM turn loop, streaming, tool execution |
| `plugin_loader.py` | Plugin discovery & loading, tier system |
| `aion_web.py` | Web UI + HTTP/SSE API |
| `prompts/rules.md` | Behavior rules for AION (loaded into system prompt) |
| `AION_SELF.md` | Technical self-documentation (on-demand via `read_self_doc`) |
| `AION_SELF_SUMMARY.md` | Auto-generated compact summary of AION_SELF.md |
| `CHANGELOG.md` | All changes chronologically |
| `MAINTENANCE.md` | This file — change propagation checklists |
| `config.json` | Runtime config (model, tier, ports, …) |
| `plugins/` | All plugins — one subfolder per plugin |
| `plugins/boot_session/` | Startup maintenance session plugin |
| `plugins/core_tools/` | Core tools: continue_work, record_mistake, lookup_rule, reflect, … |
| `plugins/heartbeat/` | Background todo worker (every 30min while running) |
| `plugins/scheduler/` | Time-based scheduled tasks |
| `plugins/proactive/` | Daily proactive analysis (08:30 weekdays) |
